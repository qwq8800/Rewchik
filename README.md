# RewchikChat Bot

Telegram-бот модерации и управления сообществом, привязанный **только** к одному чату:

- Чат: **@RewchikChat**
- Chat ID: **-1004458436938**

Если бота добавить в любой другой чат — он пришлёт сообщение о том, что настроен только на `@RewchikChat`, и сразу выйдет из чужого чата. Все команды и сообщения в других чатах полностью игнорируются.

## Реализованный функционал (по разделам ТЗ)

| Задача из ТЗ | Что реализовано |
|---|---|
| 1. Модерация | Антифлуд (**теперь покрывает не только текст, но и стикеры/фото/видео/голосовые — раньше спам медиа без подписи полностью обходил защиту**), антиспам (нечёткие дубликаты от новых участников), антиреклама (домены/стоп-слова, управляются командами `/addword`, `/adddomain` и т.д. прямо в чате), антикапс, антиупоминания, антиповторы символов, капча для новичков (математический вопрос + автокик по таймауту), лестница наказаний варн→мут→бан с автоснятием мута, срок действия предупреждений (по умолчанию 30 дней) плюс ручное управление (`/unwarn`, `/clearwarns`), устойчивость таймеров мута/капчи к рестарту процесса, антирейд — автообнаружение массового вступления новых участников за короткое окно с автоматической блокировкой чата (`/lockdown`, `/unlock`) и восстановлением исходных прав |
| 2. Вовлечение | XP/уровни за сообщения, репутация (`/rep`) с антифродом (кулдаун), топ по XP/репутации/балансу (`/top`), приветствие и прощание с участниками, экономика (монеты за сообщения, ежедневный бонус, бонус за левелап, **перевод монет между игроками `/pay`**), достижения (автовыдача по прогрессу), 4 мини-игры на ставку: `/dice`, `/coinflip`, `/slots` (все — честные 50/50 или сбалансированный house edge ~4%), `/duel` (дуэль между двумя игроками с подтверждением от соперника), магазин косметических бейджей (`/shop`, `/buy`, `/inventory`, `/equip`), кастомное отображаемое имя (`/setname`) — бот тегает участника этим именем во всех уведомлениях в чате вместо имени профиля, смена раз в 2 суток с подтверждением |
| 3. Управление | Кнопочная админ-панель `/settings`: модерация (7 переключателей), логи, пользователи, **📨 жалобы**, **гибкие роли с произвольным набором прав** (`/createrole`, `/setrole`, `/roles`), репутация, экономика, магазин, достижения, статистика, настройки |
| 4. Прозрачность | Полный лог всех действий (роли, капча, вход/выход, выдача монет админом, покупки) в БД, просмотр через панель, команда `/stats`, **система жалоб `/report` с кнопками быстрой реакции (мут/бан/отклонить) прямо под уведомлением в чате и разделом «Жалобы» в панели** |

**Осознанно не реализовано** (казино с несколькими играми, кастомные права ролей, Telegram Stars, веб-дашборд, Kafka/Redis-инфраструктура — см. разделы 6–19 исходного ТЗ). Магазин сейчас — это косметические бейджи без игровых преимуществ (чтобы не ломать баланс модерации); каркас (`shop_items`, `inventory` в БД) готов для добавления предметов с реальным эффектом.

## Установка и запуск

1. Получите токен бота у [@BotFather](https://t.me/BotFather).
2. Добавьте бота в чат **@RewchikChat** и выдайте ему права администратора:
   - удаление сообщений
   - блокировка/разблокировка участников
   - ограничение участников (мут)
3. Установите зависимости:

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

4. Скопируйте `.env.example` в `.env` и впишите реальный токен:

```bash
cp .env.example .env
# отредактируйте .env: BOT_TOKEN=ваш_токен
```

5. Запустите бота:

```bash
python3 main.py
```

Бот работает в режиме long polling — отдельный сервер/webhook не требуется, достаточно, чтобы процесс был запущен постоянно (рекомендуется через `systemd`, `pm2` или Docker для продакшена).

## Работа в личных сообщениях

Помимо группы @RewchikChat, бот отвечает на `/start` и часть информационных команд в личных сообщениях (`/rules`, `/rank`, `/setname`, `/top`, `/rep`, `/mods`, `/balance`, `/daily`, `/dice`, `/achievements`) — они не требуют прав администратора и не обращаются к контексту группы. Вся модерация, капча, вход/выход участников и админ-команды (`/settings`, `/stats`, `/warn`, `/mute`, `/kick`, `/ban`, `/promote` и т.д.) работают **только** в самой группе @RewchikChat — в личных сообщениях они игнорируются.

## Команды

Полный и всегда актуальный список команд доступен прямо в чате по команде `/help` (работает и в группе, и в личных сообщениях с ботом). Кнопочная панель — основной интерфейс для администрирования:

- `/settings` — открыть админ-панель (только для администраторов чата)
- `/help` — список всех команд с разбивкой по категориям
- `/rules` — показать правила чата
- `/rank` — свой уровень/XP/репутация/баланс
- `/setname <имя>` — задать отображаемое имя, которым бот будет тегать вас в чате (варны, приветствие, левелап и т.д.) вместо имени профиля Telegram. Требует подтверждения кнопкой, меняется не чаще раза в 2 суток
- `/top` — топ участников по XP, репутации и балансу
- `/stats` — обзор чата (только админы/модераторы)
- `/report [причина]` — ответом на проблемное сообщение отправить жалобу администраторам (кулдаун 2 мин от спама)
- `/rep` — ответом на сообщение участника начислить ему +1 репутации (раз в час на одного человека)
- `/mods` — список назначенных модераторов
- `/balance` — узнать свой баланс монет
- `/daily` — забрать ежедневный бонус монет
- `/dice <ставка>` — мини-игра: кубик 1-6, выпало 4-6 — выигрыш x2, иначе ставка сгорает (честные 50/50)
- `/coinflip <ставка> <орёл|решка>` — монетка, угадал сторону — выигрыш x2 (честные 50/50)
- `/slots <ставка>` — слот-машина: 3 одинаковых символа — выигрыш x3–x8 от ставки, 2 одинаковых — небольшой выигрыш x1.5, иначе ставка сгорает (мягкий house edge ~4%)
- `/duel <ставка>` (ответом на сообщение соперника) — вызов на дуэль; соперник должен подтвердить кнопкой, после чего деньги честно (50/50) переходят победителю. Вызов автоматически отменяется через 5 минут без ответа
- `/pay <сумма>` (ответом на сообщение получателя) — перевести монеты другому участнику
- `/shop` — список товаров магазина
- `/buy <код_товара>` — купить товар
- `/inventory` — ваш инвентарь
- `/equip <код_товара>` — экипировать купленный бейдж (отображается в /rank)
- `/achievements` — список достижений и прогресс

Ручные команды модерации (используются **ответом** на сообщение нарушителя):

- `/warn` `[причина]` — доступно администраторам и ролям с правом «moderate»
- `/mute` `[минуты]` (по умолчанию 30) — доступно администраторам и ролям с правом «moderate»
- `/unmute` — доступно администраторам и ролям с правом «moderate»
- `/unwarn` — снять последнее предупреждение — доступно администраторам и ролям с правом «moderate»
- `/clearwarns` — снять все предупреждения разом — доступно администраторам и ролям с правом «kick_ban»
- `/kick` `[причина]` — доступно администраторам и ролям с правом «kick_ban»
- `/ban` `[причина]` — доступно администраторам и ролям с правом «kick_ban»
- `/unban` — доступно администраторам и ролям с правом «kick_ban»
- `/promote` — быстрый ярлык: назначить встроенную роль «модератор» (только администраторы)
- `/demote` — снять роль модератора (только администраторы)
- `/give <количество>` — выдать монеты участнику (только администраторы)
- `/lockdown` / `/unlock` — экстренная ручная блокировка/разблокировка чата (только администраторы)

## Гибкие роли и права

Помимо `/promote`/`/demote` (быстрый ярлык для встроенной роли «модератор»), администраторы могут создавать **свои роли с произвольным набором прав**:

- `/createrole` — без аргументов покажет список доступных прав и пример. Формат: `/createrole <ключ> <название>; <право1,право2,...>`
  Например: `/createrole senior Старший модератор; moderate,kick_ban,view_stats`
- `/roles` — список всех созданных ролей и их прав
- `/setrole <ключ_роли>` (ответом на сообщение участника) — назначить роль
- `/removerole` (ответом на сообщение участника) — снять любую роль

Доступные права: `moderate` (warn/mute/unmute), `kick_ban` (kick/ban/unban), `manage_settings` (настройки, стоп-слова, домены), `view_stats` (`/stats`), `manage_reports` (кнопки под жалобами). Реальные администраторы/создатель Telegram всегда имеют все права, независимо от назначенных ролей.

## Капча для новичков

При входе нового участника (если включено в настройках) бот:
1. Временно ограничивает права нового участника (не может писать/отправлять медиа).
2. Присылает простой математический вопрос с 4 вариантами ответа на кнопках.
3. При правильном ответе — снимает ограничения и показывает приветствие.
4. Если участник не отвечает в течение заданного времени (по умолчанию 2 минуты) — бот удаляет его из чата (кик, не бан — он сможет зайти по новой ссылке).

## Антирейд

Бот отслеживает частоту вступлений в чат. Если за короткое окно (по умолчанию 60 сек) вступает подозрительно много новых участников (по умолчанию 5+) — это считается вероятным рейдом:

1. В чат отправляется тревожное уведомление.
2. Если включена автоблокировка (по умолчанию да) — бот сохраняет текущие права чата и **ограничивает возможность писать для всех, кроме администраторов**, до ручной команды `/unlock`.
3. Администратор может включить/выключить блокировку и в любой момент вручную: `/lockdown` / `/unlock`, либо через раздел «🚨 Антирейд» в `/settings`.

**Важно:** для работы блокировки/разблокировки боту нужно право **«Изменение настроек чата»** (Change chat info) при выдаче прав администратора в Telegram — без него команды `/lockdown`/`/unlock` и автоблокировка при рейде вернут ошибку.

## Логика наказаний

1-е нарушение любого анти-модуля → предупреждение (варн).
2-е и далее → мут по нарастающей лестнице (по умолчанию 5 мин → 30 мин → 3 часа → 1 день), после исчерпания лестницы — бан.
Лестницу можно изменить в `config.DEFAULT_SETTINGS["punishment_ladder"]` (в секундах, через запятую) до первого запуска, либо позже — через прямое обновление таблицы `settings` в БД.

## Деплой на Railway

Бот работает через long polling — публичный порт/домен не нужен, деплоится как обычный worker-процесс.

### 1. Подготовка репозитория

Загрузите содержимое этой папки в GitHub-репозиторий (или используйте `railway up` из локальной папки без GitHub — см. вариант Б ниже). В проекте уже есть всё нужное для Railway:

- `Procfile` — команда запуска (`worker: python main.py`)
- `railway.json` — конфигурация билда/деплоя (Nixpacks, `ON_FAILURE` рестарт)
- `nixpacks.toml` — фиксирует Python 3.12 и `pip install -r requirements.txt`
- `.gitignore` — исключает `.env`, `bot.db`, `venv/`

### 2. Вариант А — через GitHub

1. Зайдите на [railway.app](https://railway.app) → **New Project** → **Deploy from GitHub repo**.
2. Выберите репозиторий с ботом.
3. Railway автоматически определит Python-проект через Nixpacks и запустит `python main.py` (по `railway.json`/`Procfile`).

### 3. Вариант Б — через Railway CLI (без GitHub)

```bash
npm install -g @railway/cli
railway login
cd rewchik_bot
railway init
railway up
```

### 4. Переменные окружения

В настройках проекта (**Variables**) добавьте:

| Переменная | Значение |
|---|---|
| `BOT_TOKEN` | токен от @BotFather |
| `DB_PATH` | `/data/bot.db` (см. следующий пункт про Volume) |
| `ALLOWED_CHAT_ID` | *(необязательно)* переопределяет chat_id рабочего чата, по умолчанию `-1004458436938` |
| `SUPER_ADMIN_ID` | *(необязательно)* переопределяет id главного администратора бота, по умолчанию `8375758522` (@Luvkryacan) |

### 5. Обязательно: постоянное хранилище (Volume)

Файловая система Railway эфемерна — при каждом новом деплое файл SQLite (`bot.db`) будет удалён, и бот "забудет" варны/муты/логи/настройки. Чтобы этого избежать:

1. В проекте на Railway откройте сервис бота → вкладка **Volumes** → **New Volume**.
2. Укажите mount path: **`/data`**.
3. Убедитесь, что переменная `DB_PATH=/data/bot.db` задана (см. п.4). Код в `config.py` и так по умолчанию переключится на `/data/bot.db`, если директория `/data` существует, но явно задать переменную — надёжнее.

### 6. Важно: сервис должен быть без публичной сети

Так как бот не поднимает HTTP-сервер (только long polling к Telegram), **не** генерируйте Railway-домен для этого сервиса и не привязывайте `$PORT` — просто оставьте сервис приватным (Networking → Public Networking выключен). Это нормально для worker-процессов.

### 7. Логи и рестарты

- Логи бота смотрите во вкладке **Deployments → View Logs**.
- `railway.json` настроен на автоматический рестарт при падении процесса (`ON_FAILURE`, до 10 попыток) — это подстраховка, аналог "продолжает базовую модерацию даже при деградации" из п.0.3 ТЗ (в рамках возможностей single-instance деплоя).
